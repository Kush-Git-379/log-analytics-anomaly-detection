# Log Analytics and Anomaly Detection Tool

A hybrid **Bash + Python** system monitoring pipeline that collects real-time resource metrics, detects statistical anomalies, and dispatches automated email alerts. Built as a course project for Scripting Language (3EC504ME24), B.Tech ECE, Nirma University — March 2026.

---

## How It Works

The system is split into two modules, each using the best tool for the job:

```
[Linux Kernel Utilities]
        ↓
  collector.sh  (Bash)
  Extracts CPU, Memory, Disk metrics → system_health.csv
        ↓
  detector.py   (Python)
  Reads CSV → computes µ + 2σ threshold → flags anomalies → sends email alert
```

**Module 1 — `collector.sh` (Data Extraction)**
- Interfaces with Linux kernel utilities (`top`, `free`, `df`) via pipelines
- Extracts CPU usage, memory usage (%), and disk usage (%)
- Appends timestamped readings to a structured CSV file
- Designed to be scheduled via `cron` for automated periodic collection

**Module 2 — `detector.py` (Anomaly Detection)**
- Reads the CSV using `pandas`
- Computes a **dynamic threshold: µ + 2σ** from historical baseline data
- Flags the latest reading as an anomaly if it exceeds the threshold
- Falls back to a hard rule-based limit (80%) for cold-start conditions

## Setup and Usage

### Requirements
- Linux / Ubuntu (Bash + standard utilities: `top`, `free`, `df`, `bc`)
- Python 3.x
- pandas (`pip install pandas`)

### Step 1 — Run the data collector
```bash
chmod +x collector.sh
./collector.sh
```
This creates `system_health.csv` in `~/log_project/` with the first reading.

### Step 2 — Schedule automatic collection via cron
```bash
crontab -e
```
Add this line to collect metrics every minute:
```
* * * * * /path/to/collector.sh
```

### Step 3 — Run the anomaly detector
```bash
python3 detector.py
```

**Sample output (normal):**
```
--- SYSTEM HEALTH REPORT : 2026-03-01 14:32:05 ---
Current CPU: 12.4% (Threshold: 45.23%)
✅  System Status: NORMAL
----------------------------------------
```

**Sample output (anomaly detected):**
```
--- SYSTEM HEALTH REPORT : 2026-03-01 14:35:12 ---
Current CPU: 87.6% (Threshold: 45.23%)
⚠️  ALERT: CPU Usage is abnormally high!
----------------------------------------
```

## Anomaly Detection Logic

The detector uses **statistical thresholding** rather than fixed limits:

```
threshold = µ + 2σ
```

Where µ is the mean CPU usage and σ is the standard deviation computed from all historical readings. This means the threshold adapts to the system's normal operating conditions — a server that normally runs at 60% CPU will have a higher threshold than a mostly-idle machine.

A hard floor of 50% is enforced during cold start (when insufficient baseline data exists).

**Known limitation:** The µ + 2σ threshold becomes accurate only after sufficient baseline data is collected. Early runs on sparse data may produce false positives.

The system was validated by simulating CPU spikes using background stress processes:

The anomaly detector successfully identified the spike and triggered the alert pipeline.
